## Before making the repository public

- [ ] Confirm the chosen license is intentional (`AGPL-3.0-only`)
- [ ] Review screenshots and docs for anything private or local-path-specific
- [ ] Verify Cloudflare Pages project name and deployment instructions
- [ ] Confirm the README still matches the current UI and product scope
- [ ] Decide whether `AGENTS.md` should remain public as a contributor-facing scope document

## Recommended first-day repo settings

- Enable Issues
- Enable Discussions only if you want product/UX conversation in-repo
- Enable branch protection on `main` if you expect external contributions
- Require PRs for direct changes if the project will accept outside contributors
